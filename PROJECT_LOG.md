# PROJECT_LOG — мозг проекта

---

## 1. О проекте в двух словах

- **Название проекта / репозитория**: FindOrigin
- **Краткое описание**: Telegram-бот для проверки новостей и поиска исходных источников. Принимает текст или ссылку на пост в Telegram, ищет похожие материалы (DuckDuckGo), с помощью AI (OpenRouter) подбирает 1–3 наиболее вероятных источника по смыслу и возвращает ссылки с оценкой уверенности.

---

## 2. Текущий статус

- **Что уже работает**: Next.js-проект с webhook, приём сообщений, извлечение текста (в т.ч. по ссылке на пост), извлечение сущностей, поиск кандидатов (DuckDuckGo), ранжирование через OpenRouter, отправка ответа в чат; локальный long-polling бот для отладки; тесты (Jest); Telegram Mini App (страница `/tma`, API `/api/tma/search`), кнопка меню в боте открывает TMA; история запросов в TMA (localStorage, до 30 записей); оформление результата — карточки источников и сводный блок.
- **Что сейчас в фокусе**: Работа по текущему этапу завершена. Дальше — по необходимости итеративные улучшения (промты AI, UX).

## 4. Технические решения и грабли

- **Рабочие решения**
  - Поиск без ключей: DuckDuckGo (парсинг HTML); опционально Google CSE при наличии ключей.
  - AI через OpenRouter (модель задаётся в .env); ответ модели — JSON с sources и confidence.
  - Webhook отдаёт 200 быстро; тяжёлую обработку делает processUpdate (поиск + AI), ответ пользователю — асинхронно через sendMessage.
  - Единая логика в `processUpdate`: используется и в webhook, и в local-bot (long-polling).
  - TMA: история запросов в localStorage (ключ findorigin_tma_history), без бэкенда; кнопка меню бота в BotFather указывает на URL /tma.
